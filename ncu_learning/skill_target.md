# NCU 学习深度标准（L1 ~ L3）

> 目标：明确“学到什么程度就够投 AI Infra 实习”

---

## L1：会用工具（入门）

你需要做到：

- 能运行 `ncu --set default ./xxx`
- 能打开/导入 `.ncu-rep` 报告
- 知道 `LaunchStats`、`Occupancy`、`SpeedOfLight` 在看什么

你需要产出：

- 1 份 kernel 的基础 profile 结果截图或数据记录

如果只到 L1：**不够面试竞争力**（容易停留在“我用过工具”）。

---

## L2：能定位瓶颈（建议最低投递标准）

你需要做到：

- 能判断瓶颈类型：Memory / Compute / Latency
- 能解释 2~3 个关键指标与瓶颈的关系
- 能给出有依据的优化方向（不是拍脑袋）

常见可解释指标（不要求背全）：

- `gpu__compute_memory_throughput.*`
- `sm__throughput.*`
- `gpu__dram_throughput.*`
- Occupancy / 寄存器 / shared memory 限制

你需要产出：

- 至少 2 个版本对比（例如 V1 vs V2）  
- 每个版本一句话瓶颈结论 + 证据指标

---

## L3：能做优化闭环（强烈建议达到）

你需要做到：

1. 先 profile 得到瓶颈证据  
2. 修改 kernel（如 coalescing / vectorized / register tiling）  
3. 再 profile 验证收益  
4. 能解释“为什么变快”

你需要产出：

- 一张 V1~V6 对比表（时间、GFLOPS、关键指标）
- 一份 1 页以内“性能优化报告”
- 一段可用于简历和面试复述的项目描述

---

## 对你当前项目的目标建议

结合你的 `final_project` 规划，建议目标：

- **必须达到：L2**
- **最好达到：L3（投递前完成）**

原因很简单：你做的是 GEMM 多版本优化路线，天然适合展示“性能工程能力”，这是 AI Infra 简历里最有区分度的部分。

---

## 面试自检（通过 5/6 即达标）

- [ ] 我能说清楚 V1 当前主要瓶颈是什么
- [ ] 我能解释 V2 为什么比 V1 快（从指标角度）
- [ ] 我能给出 V3/V4 的优化假设，并设计验证方法
- [ ] 我知道 Occupancy 低不一定是性能差的唯一原因
- [ ] 我能把一次优化讲成“证据链”
- [ ] 我能用 3 分钟讲完一个版本升级故事
